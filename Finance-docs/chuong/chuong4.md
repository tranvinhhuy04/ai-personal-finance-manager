# CHƯƠNG 4: CÔNG NGHỆ SỬ DỤNG

## Bảng 4.1. Công nghệ chủ chốt và lý do lựa chọn

| STT | Thành phần | Công nghệ | Lý do chọn |
|---|---|---|---|
| 1 | Frontend | React 19, Vite 6, TypeScript 5.8, Tailwind CSS 4, React Query v5, Zustand v5, Recharts v3 | React 19 với mô hình concurrent rendering phù hợp cho giao diện tài chính có mật độ dữ liệu cao và cập nhật thời gian thực. Vite 6 rút ngắn thời gian khởi động môi trường phát triển so với Webpack truyền thống. React Query đảm nhận quản lý server state (fetch, cache, invalidation) theo cơ chế khai báo, tách biệt rõ ràng với client state do Zustand quản lý. Recharts cung cấp bộ biểu đồ SVG có khả năng tích hợp trực tiếp vào JSX mà không phụ thuộc thư viện DOM ngoài. |
| 2 | Backend | Node.js/Express (TypeScript), Mongoose v8, jsonwebtoken v9, bcryptjs v3, decimal.js v10, amqplib v0.10, Docker Compose | Node.js phù hợp với đặc tính I/O-bound của các microservice tài chính nhờ mô hình event loop phi chặn (non-blocking). `decimal.js` được sử dụng thay cho kiểu `Number` nguyên thủy để loại trừ sai số dấu phẩy động trong các phép tính số dư — yêu cầu bắt buộc với dữ liệu tiền tệ. JWT stateless giảm sự phụ thuộc giữa các service; Docker Compose đảm bảo môi trường triển khai nhất quán trên mọi máy chủ. |
| 3 | AI / Agentic RAG | Google Gemini 2.5 Flash, LangChain Google GenAI v2.1, FastAPI v0.115, Redis (cache context 30 s, session 120 s) | Gemini 2.5 Flash được lựa chọn nhờ cửa sổ ngữ cảnh lớn và độ trễ thấp, phù hợp với yêu cầu chatbot tài chính phản hồi thời gian thực. LangChain cung cấp lớp trừu tượng cho kiến trúc Agentic RAG, cho phép thay thế mô hình LLM mà không ảnh hưởng đến luồng điều phối. FastAPI với Pydantic và async I/O xử lý các yêu cầu AI đồng thời hiệu quả hơn framework WSGI đồng bộ. Cơ chế API Key Pool (tối đa 10 khóa, mã hóa AES-256-CBC, tự động luân chuyển khi nhận lỗi HTTP 429) nâng cao tính sẵn sàng của dịch vụ AI. |
| 4 | NLP | PhoBERT (`vinai/phobert-base`), HuggingFace Transformers v4.46, SentencePiece v0.2 | PhoBERT là mô hình BERT được tiền huấn luyện trên 20 GB văn bản tiếng Việt, cho độ chính xác phân loại ý định (intent classification) vượt trội so với các mô hình đa ngôn ngữ tổng quát trong miền tài chính tiêu dùng tiếng Việt. SentencePiece xử lý chính xác đặc trưng từ ghép và dấu thanh của tiếng Việt mà whitespace tokenizer thông thường không đáp ứng được. |
| 5 | OCR | PaddleOCR v2.9 (`lang=vi`, `use_angle_cls=True`), PaddlePaddle v2.6, OpenCV v4.10 | PaddleOCR cho độ chính xác nhận dạng văn bản tiếng Việt có dấu trên ảnh hóa đơn thực tế cao hơn Tesseract, đặc biệt với ảnh chụp từ điện thoại có nhiễu hoặc góc nghiêng. Tham số `use_angle_cls=True` tự động hiệu chỉnh góc xoay tài liệu. Mô hình được khởi tạo theo mẫu Singleton thread-safe (double-checked locking) để tránh tải lại mô hình (~2–3 giây/lần), giảm độ trễ từ request thứ hai trở đi. |
